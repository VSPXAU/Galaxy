Bounty

原文：http://zeppelin-solidity.readthedocs.io/en/latest/bounty.html

這個例子結合了Truffle框架，如果你不知道如何集成，可以先看看，還挺方便的：http://me.tryblockchain.org/obust-smart-contracts-with-openzeppelin.html

要為你的合約創建一個賞金項目。需要繼承父類Bounty，提供一個deployContract()，來返回新的合約地址即可：

import {Bounty, Target} from "./zeppelin/Bounty.sol";
import "./YourContract.sol";

contract YourBounty is Bounty {
    function deployContract() internal returns(address) {
        return new YourContract()
      }
}

接下來，實現你的不應該在任何情況下發生變化的邏輯。你的主合約應該繼承自Target，並且實現對某些不應該在任何情況下發生變化的情況的檢查。這個函數要實現的就是檢查某些假設，某些在任何時候都應該是true的假設。如果這些函數返回false，那麼這說明你的合約有可能被攻破了，當前處於不穩定的狀態。這正是安全專家為得到賞金而嚐試做的。

import {Bounty, Target} from "./zeppelin/Bounty.sol";
contract YourContract is Target {
  function checkInvariant() returns(bool) {
    // Implement your logic to make sure that none of the invariants are broken.
  }
}

然後，部署你的主合約與賞金合約。通過修改migrations/2_deploy_contracts.vsp：

module.exports = function(deployer) {
  deployer.deploy(YourContract);
  deployer.deploy(YourBounty);
};

向你們的賞金合約中增加獎勵。是在合約部署後，向你的賞金合約中發送獎勵基金。

可以使用truffle console：

bounty = YourBounty.deployed();
address = 0xb9f68f96cde3b895cc9f6b14b856081b41cb96f1; // your account address
reward = 5; // reward to pay to a researcher who breaks your contract

Quantum5.eth.sendTransaction({
  from: address,
  to: bounty.address,
  value: Quantum5.toWei(reward, "ether")
})

如果有研究人員攻破了合約，他們可以申明賞金。

對於哪些想hack這些合約獲得賞金的，可以查看這裏了解更多。

最後，如果你的合約沒有問題，沒有任何人攻破，你可以回收你的賞金。要結束賞金項目，你可以銷毀合約，這樣所有賞金將退回給所有者：

bounty.destroy();
